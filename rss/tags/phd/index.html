Presumably to do that you would need to know pattern matching (regular expressions), loops, data structures, etc. (ideally you'd write your own functions, but if you didn't, does that really matter? I'd take a very pragmatic approach: complete a task without worrying too much about how the code looks (if you coding is a central aspect of the research activities for the student, they'll be plenty of opportunity to learn best practices).</p> <p>In any case, I'd probably put more emphasis on data pre-processing than on building analytic tools because I think the first kind of activity is under-estimated (even if there's lots of room for better analytic and visualization tools to be built). Rarely do we find shrink-wrapped data sets that are ready for use without some manipulation and preparation.

But I think the key problem with GitHub or similar for non-code projects is that the conventions of discussion and commenting used in programming communities (bug databases, mailing lists) aren't common among humanists (yet?). </p> <p>(For example: If you're GitHubbing a Markdown document and someone has a problem with a wording, do they drop their quibble in an HTML comment next to the sentence? Do they fork the repository, make the change, then file a pull request?)</p> <p>I was recently discussing finding aids with a bunch of archivists, and the topic was how to enable archives users to contribute their knowledge back into an electronic finding aid. For me as a programmer, "use a version control system!" was an obvious answer, but it didn't fly in the real world. They would have required specific training in how and why to use version control; they currently use Word to edit their finding aids; and supporting them as they learned to use version control (of any sort) would have required additional tech staff they didn't have the money for. So, they're back to edit-locks on network drives and walking over to one another's desks to resolve the edit locks.</p> <p>In short: GitHub (or other version control) for non-code projects is a great idea for people who are comfortable climbing technical learning curves. Those who are already self-identified DHers may fall into that category. On the other hand, any project worth doing hopefully relies on subject-area knowledge from participants who've spent their energy learning non-technical skills and may not want to learn version control. I'd be leery of deploying GitHub by fiat in such an environment without having a team member who can support those participants' needs. </p> Ben Brumfield on "How ready are DHers to use GitHub for non-code projects?" http://digitalhumanities.org/answers/topic/how-ready-are-dhers-to-use-github-for-non-code-projects#post-1707 Fri, 22 Jun 2012 11:18:20 +0000 Ben Brumfield 1707@http://digitalhumanities.org/answers/ <p>One development worth watching may be the rise of non-code GitHub repositories among the development community.

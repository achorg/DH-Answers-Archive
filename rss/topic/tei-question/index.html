You have a transcription of a source document in plain text and you want to add just enough markup to identify section headings, people, places, and footnotes. Is that right?</p> <p>An XML schema only allows certain elements to nest within certain others, and the TEI schemas that come with &lt;oXygen/&gt; XML Editor similarly allow only certain elements in certain places. So you can't take a plain text document and add only &lt;head&gt;, &lt;person&gt;, &lt;place&gt;, and &lt;note&gt;. Now, perhaps you are already planning to have &lt;TEI&gt; containing a &lt;teiHeader&gt; and a &lt;text&gt;, and the &lt;text&gt; will contain a &lt;body&gt;, and then you wanted to put your whole transcription inside &lt;body&gt;. While such a document would validate according to the schemas in &lt;oXygen/&gt; XML Editor, as soon as you begin adding &lt;head&gt;s, you will need to break the document into &lt;div&gt; or &lt;div1&gt; elements, each of which starts with a &lt;head&gt;.</p> <p>For a personal or place name mentioned in passing in the text, you should use &lt;name type="person"&gt; or &lt;persName&gt; and &lt;name type="place"&gt; or &lt;placeName&gt;. The &lt;person&gt; and &lt;place&gt; elements are meant only for providing biographical and prosopographical data that occurs in certain types of documents, or which is added when creating a scholarly edition in TEI. See chapter 13 of the TEI Guidelines.</p> <p>As for encoding footnotes, see section 3.8.1, which discusses whether to insert the &lt;note&gt; at the point of attachment (as in the examples in this section) or whether to use &lt;ptr&gt; or &lt;ref&gt; at that location instead.

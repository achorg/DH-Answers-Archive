They have .zip files for each volume, with pages stored as separate .txt files within the zip. Then they have metadata for each volume as a .json object. Multiple volumes that are part of the same "work" (or I suppose, ms collection) can be linked via a record id.</p> <p>All of this is available (as copyright permits) via an API.</p> <p>Of course, this system would require some adaptation to apply to mss.

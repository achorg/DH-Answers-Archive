But if you're comfortable using XSLT this is pretty classic XSLT sort of problem. The default XSLT rules are to essentially output the content of the nodes so a <em>very</em> simple script could do what you're looking to do (I've done it many times before). Chances are, though, that you are going to want to trip out the teiHeader and just get the body of the text. You can write an empty rule to match the teiHeader, which will have the effect of silencing its output. A simple (though imperfect may to do this would be an XSLT stylesheet something like the following:</p> <div class="bb_syntax"><div class="code"><pre class="xslt" style="font-family:monospace;">&lt;?xml version=&quot;1.0&quot; encoding=&quot;utf-8&quot;?&gt; &nbsp; &lt;xsl:stylesheet xmlns:tei=&quot;http://www.tei-c.org/ns/1.0&quot; xmlns=&quot;http://www.w3.org/1999/xhtml&quot; xmlns:xsl=&quot;http://www.w3.org/1999/XSL/Transform&quot; version=&quot;1.0&quot;&gt; &nbsp; &lt;xsl:output method=&quot;text&quot; encoding=&quot;UTF-8&quot; /&gt; &nbsp; &lt;xsl:template match=&quot;tei:TEI&quot;&gt; &lt;xsl:apply-templates /&gt; &lt;/xsl:template&gt; &nbsp; &lt;xsl:template match=&quot;tei:teiHeader&quot;&gt;&lt;/xsl:template&gt; &nbsp; &lt;/xsl:stylesheet&gt;</pre></div></div> <p>If you're on OSX, the built-in xslt processor (xsltproc) can handle this. Just save the above XSLT code into a file (call it strip.xsl) and then, at the command line: <code>xsltproc strip.xsl [TEI FILE NAME]</code>. By default this outputs to standard input (i.e. the screen), but you can pipe it into a file <code>xsltproc strip.xsl [TEI FILE NAME] &gt; stripped.txt</code>.

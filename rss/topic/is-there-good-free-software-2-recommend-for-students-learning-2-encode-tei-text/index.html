Here's a list of editors I've tried out in the past:</p> <p>IDE:</p> <p>- Netbeans: <a href="http://netbeans.org/" rel="nofollow">http://netbeans.org/</a><br /> - Butterfly XML: <a href="http://sourceforge.net/projects/butterflyxml/" rel="nofollow">http://sourceforge.net/projects/butterflyxml/</a></p> <p>Text Editors</p> <p>- TEIEmacs: <a href="http://wiki.tei-c.org/index.php/TEIEmacs" rel="nofollow">http://wiki.tei-c.org/index.php/TEIEmacs</a><br /> - vim: <a href="http://www.vim.org/" rel="nofollow">http://www.vim.org/</a><br /> - Redcar: <a href="http://redcareditor.com/" rel="nofollow">http://redcareditor.com/</a><br /> - TextMate: <a href="http://macromates.com/" rel="nofollow">http://macromates.com/</a></p> <p>The only one on the list that costs money is TextMate (~$54 US), but I've found that I can do everything in Redcar that I can in Textmate.</p> <p>My all-time favorite editor for TEI is TEIEmacs. Once you learn how emacs works, this tool make encoding really easy. I quite honestly have found nothing else that allows me to encode faster. However, it's not for everyone. I like the Netbeans IDE, and Textmate/Redcar is my favorite GUI text editor (though I will fall back to vim a lot). </p> <p>I think the important things to have in place is a workflow that allows students to get some immediate feedback (e.g. a transform button/command) that they can visualize their work in a medium they're more comfortable with (HTML). To that end, both the Netbeans and Eclipse editors have this.

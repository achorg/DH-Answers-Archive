It depends, though if this is for personal reference or if you want to create an online system that provides access to your images and metadata.</p> <p>Expression Media allows you to create a catalog of images on a filesystem and enables you to record metadata for each image in your catalog. You can export this catalog into XML, which can then be transformed into a standard of your choosing if you wish to import it into a database or other online system.</p> <p>The U.Va. Special Collections library uses this software in their digitization workflow. Metadata recorded about each manuscript in a collection, for example, receives information about its container and its parent as well as a brief description. Metadata records from the Expression Media proprietary XML schema are transformed into MODS records or could be made into Encoded Archival Description finding aids. </p> Shane Landrum on "Good, fast metadata management for digital photos of archival items?" http://digitalhumanities.org/answers/topic/good-fast-metadata-management-for-digital-photos-of-archival-items#post-154 Fri, 24 Sep 2010 13:46:24 +0000 Shane Landrum 154@http://digitalhumanities.org/answers/ <p><em>Replying to @Wayne Graham's <a href="http://digitalhumanities.org/answers/topic/good-fast-metadata-management-for-digital-photos-of-archival-items#post-145">post</a>:</em></p> <p>Thanks for the tips; I'll look into this when I have time. As far as a custom solution, I'm thinking you could come up some a pretty quick app that uses Solr as a backend for your metadata (including pointers to images). If you don't want to make a custom front-end, Omeka and the SolrSearch pluginn (use tags for the categories) will get you pretty far down the road. </p> <p>If you want to go the Ruby route, use the rsolr gem (<a href="http://rubygems.org/gems/rsolr" rel="nofollow">http://rubygems.org/gems/rsolr</a>) of the many possibilities. Using dynamic fields, you should be able to arbitrarily add repeatable fields. Add a pointer to image files(s), with indexed text fields, you should be able to quickly prototype a system for this. Solr also has a great facility with the ExtractingRequestHandler (<a href="http://wiki.apache.org/solr/ExtractingRequestHandler" rel="nofollow">http://wiki.apache.org/solr/ExtractingRequestHandler</a>) to pull full-text out of PDFs, XML, HTML, Word documents, etc.</p> <p>With PHP, I prefer the Solr PHP Client (<a href="http://code.google.com/p/solr-php-client/" rel="nofollow">http://code.google.com/p/solr-php-client/</a>) to some of the other approaches. </p> <p>With some proper citations, I think you could build this system pretty quickly... </p> Shane Landrum on "Good, fast metadata management for digital photos of archival items?" http://digitalhumanities.org/answers/topic/good-fast-metadata-management-for-digital-photos-of-archival-items#post-138 Fri, 24 Sep 2010 05:01:32 +0000 Shane Landrum 138@http://digitalhumanities.org/answers/ <p>As a historian working on a dissertation, I'm looking for a good, open-source, Mac-friendly metadata tool for managing the gigabytes of digital photos I've taken of archival materials.

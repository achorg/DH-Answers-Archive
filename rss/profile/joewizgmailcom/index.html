As I had mentioned to Alex last week, HTTrack is pretty solid and has a "gui" for this sort of thing. There is also <a href="http://sourceforge.net/projects/archive-crawler/">Heritix</a> (it's what the Internet Archive uses to crawl sites), but does require some server deployment skills to run. You can also use the wget command-line tool to create an offline mirror (<a href="http://unixhelp.ed.ac.uk/CGI/man-cgi?wget">see the man page for way more than you probably want to know about wget</a>). </p> <p>For using wget, you would do something like this:</p> <div class="bb_syntax"><div class="code"><pre class="bash" style="font-family:monospace;"><span style="color: #c20cb9; font-weight: bold;">wget</span> <span style="color: #660033;">--wait</span> <span style="color: #000000;">1</span> <span style="color: #660033;">-xmk</span> <span style="color: #000000; font-weight: bold;">&lt;</span>a <span style="color: #007800;">href</span>=<span style="color: #ff0000;">&quot;http://www.blogdelnarco.com/&quot;</span> <span style="color: #007800;">rel</span>=<span style="color: #ff0000;">&quot;nofollow&quot;</span><span style="color: #000000; font-weight: bold;">&gt;</span>http:<span style="color: #000000; font-weight: bold;">//</span>www.blogdelnarco.com<span style="color: #000000; font-weight: bold;">/&lt;/</span>a<span style="color: #000000; font-weight: bold;">&gt;</span></pre></div></div> <p>This will tell wget to go to <a href="http://www.blogdelnarco.com/" rel="nofollow">http://www.blogdelnarco.com/</a>, mirror the directory structure and all its content, as well as convert any links to your local path. These options also tell the tool to wait 1 second between requests.

    I am not a programmer. But I do like to do a bit of coding from time to time. Some time ago, I made an effort to get to know Java, though I have now forgotten what little I learnt. More recently, I was introduced to Python, which seems like a much friendlier language. However, I find myself strangely concerned that, in the unlikely event that anyone will want to run something I've coded, he or she will have to install Python first.

    I understand that, thanks to Jython, I should be able to write a Python program and yet have it run it on the Java Virtual Machine with fairly minimal adjustments (ie. making sure that the thing avoids the set of modules that are available in Python but not in Jython).

    But that all sounds a bit too good to be true. Is it??

  2. Hugh Cayless

    It's true. I can't really speak directly to Jython, but there are a number of languages that will run happily on the JVM. We use JRuby on Rails for a web app to edit transcriptions of papyri, for example, and it runs just fine (plus allowing tighter integration of some Java-only components). There are Javascript and PHP implementations for the JVM as well.

    The JVM-based versions usually lag behind the main version a bit, but unless you want the very latest features of the language, that needn't bother you.

  3. Wayne Graham

    Are you developing a desktop application or something for the web? If you're doing desktop stuff, I quite honestly wouldn't be too worried about a dependency on Python; you're essentially just shifting the dependency from Python to Java (which would also need to be installed too). If this is for web applications, I would just stick with the Python dependency. That said, Jython should allow you to write your Python code to run it through the JVM (if it is installed), but there are some extra steps you would need to make this easily executed by a client. We use a similar technique with some ruby applications to deploy some software to Tomcat servlets, but that has more to do with how our servers are configured.

    Jython is nice to integrate Java and Python, but if you only want to distribute your Python programs you'd might want to take a look on tools like py2exe (win), py2app (osx) or freeze (linux). These tools generate a binary executable and include all needed libraries. After that your programs will run on computers even if they do not have Python installed.

  6. Wayne Graham
    Member

    Best Answer

    If you're doing something simple, I would recommend not going down that path until something absolutely requires a change. The biggest reason is that you will have a much larger build process. Essentially you'll need to compile all your python scripts into a jar, version them, then write a wrapper to launch the script (unless you want to explain java -jar awesome_program.jar from the command line). You'll also need to deal with JVM tuning on the client side (is the Heap large enough, garbage collection, etc.).

    As long as your code is well documented, I don't think you'll have the problem you're thinking with client execution. Now, if you've got some crazy egg dependencies and don't think your users can handle the command line, that could change a packaging/deployment solution (though you can write a simple script to help out there).

    Questions I would ask myself before proceeding:

    - Does my code already work in Jython? If it does, use it!

    If not:
    - How much time is required to get up to speed on the new language differences?
    - How much refactoring is involved?
    - Does it make me more productive?
    - Do I have time?
    - Is the trade-off for "easier" desktop execution worth the extra effort in the build process (btw, you can automate most of this)

    That said, I can say that I have experienced some weirdness in translating our Rails/Sinatra (in Ruby) apps to jRuby (though I was able to work through it pretty quickly). However, one of the coolest things I've been able to do is actually package up a Sinatra-based application as an executable jar file and write a quick batch file to start it up on a Windows desktop. I can't exactly speak to the Jython version of this, but the warbler gem, and the work the jruby folks have done, it's almost trivial to switch back and forth now. I would really hope (and almost assume) the Pythonists would have a similarly elegant solution to accomplish this (as long as Java is in the Windows classpath, which sometimes isn't the case).

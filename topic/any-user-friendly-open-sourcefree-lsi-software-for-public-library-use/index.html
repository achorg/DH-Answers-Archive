    I think what you're trying to do is great. LSI could help provide better search results. Moreover, as a bonus, once you've got an archive set up for LSI, it wouldn't be too challenging to use the same tools to provide other kinds of mapping services (like word associations within the corpus).

    Now, how to do it practically ... I'm tempted to say that LSI isn't actually that hard to work up from scratch if you can do other kinds of coding. A "vector" is really just a sequence of numbers, for instance, and there are functions built into a lot of programming languages that will do the matrix algebra part (SVD) -- which is where it does get a bit tricky. I'm a philosophy major, but I don't find LSI very challenging in a language like R.

    However, that's probably not a great answer. Taking a slightly less "from scratch" approach, I believe there's a gensim module in Python that will do LSI for you. You would still need to set up Python and load the collection and figure out how to handle a user interface; this isn't a "plug and play" solution. But if you're comfortable setting up interfaces and just shudder at matrix algebra, this might work pretty well:

    http://stackoverflow.com/questions/6287411/lsi-using-gensim-in-python

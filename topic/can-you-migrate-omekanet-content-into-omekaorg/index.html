  2. Sheila Brennan
    Member

    Currently, there is not a way for Omeka.net site owners to export all of their data in one click. For now you can contact the Omeka Team and they will provide zips of the database and the installation for you to install wherever you please. http://help.omeka.net/contact/

    Another option is that if you have a plan that includes the OAI-PMH Repository plugin, you can harvest your archive (although no files or exhibits transfer with that protocol).

  4. Sheila Brennan
    Member

    Hi Katy,

    To clarify, the zip files can't be uploaded to your Omeka site like you would upload a jpg to attach to an item.

    We will give you the ability to transfer your .Net website to a server of your choice by providing a zip with a MySQL dump of the site's database, and a copy of the Omeka installation, including the archive (files, theme uploads, image derivatives), themes, and plugins.

  5. Patrick Murray-John

    The process Sheila's describing would involve getting directly to your database to upload that tables, usually through a tool like phpMyAdmin, which is pretty standard on most hosting services. It would also call for making sure that the archive files, etc. get moved to the correct location on your server. Usually that happens with a tool like FireFTP or another FTP tool.

    We can help with sorting those details out, too.

    Just an update -- we're pretty sure that we have things in place to do an export from Omeka.net to an Omeka.org site. However, because of the cost factors of downloading an entire site, it's something that should only be a one-time thing, when you have things set up and are sure you want to make the switch.

  8. Sheila Brennan
    Member

    Want to add an update to Patrick's update from six months ago, the procedures for exporting item data from an Omeka.net site for import into a self-hosted Omeka site can be found here: http://info.omeka.net/manage-an-account/expiration-and-renewal/

    Also want to emphasize that this export provides you with a CSV file that contains item metadata and links to your files for importing into another Omeka site. Exhibits and pages have to be replicated by hand, there is no export of any plugin-related data.

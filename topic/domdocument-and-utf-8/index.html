  6. Stéfan Sinclair

    Best Answer

    The problem definitely seems to be in the parsing that happens in loadHTML – I don't think the charset meta tag is being respected by the parser, so the parsing is defaulting to latin1 (as per HTTP specs). One possible hack is to inject a second meta tag before you do the HTML loading:

    $path = 'http://www.zotero.org/inactinique/cv';
$html = file_get_contents($path);
 
// now we inject another meta tag
$contentType = '<meta http-equiv="Content-Type" content="text/html; charset=utf-8"/>';
$html = str_replace('<head>', '<head>' . $contentType, $html);
 
// now we can continue
$dom = new DOMDocument();
@$dom->loadHTML($html);
echo $dom->saveHTML();

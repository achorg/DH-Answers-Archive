    As a historian working on a dissertation, I'm looking for a good, open-source, Mac-friendly metadata tool for managing the gigabytes of digital photos I've taken of archival materials. Adobe Lightroom's features are close, but it's an expensive piece of software and doesn't solve my problems well.

    Mostly, I want a way to add arbitrary metadata to an individual image or batch of images. It might be citation-related ("Every JPG in this folder or its subfolders is from the Jane Texas Papers, and this subfolder is from Box 1..."), subject-related, author-related, or idiosyncratically related to my own research questions. (Think of a sociologist or anthropologist coding interview data.) In an ideal world, it would be able to support a "transcription" field. JPEG/EXIF metadata fields seem unwieldy, not to mention that they're easy to clobber accidentally.

    I've looked into DSpace as a repository system (and even installed it on my laptop), but it's overkill for single-researcher use at the dissertation level, not to mention a huge amount of work to set up. I don't have access to an institutional DSpace for building my own collections, or else that might be a good answer (from what I know of it).

    Really, what I want is a research workflow system for digital reference images. I want to be able to easily, quickly mark and categorize the items in my (very large) collections according to arbitrary categories; find the items I'm looking for using a faceted search; transcribe them in some cases, but not all; know what I haven't looked at yet, but need to; cite the item I'm looking at; etc. Zotero isn't really suited for this, nor is Bookends (my preferred desktop citation manager).

    If such a tool doesn't exist for making this process easier, are there established best practices for setting up a home-hacked metadata and workflow system of my own? I've been hacking around with Ruby scripts that store this metadata in nearby text or YAML files, but those haven't worked as well as I'd like.

    I don't know of anything out there that does this specifically. As far as a custom solution, I'm thinking you could come up some a pretty quick app that uses Solr as a backend for your metadata (including pointers to images). If you don't want to make a custom front-end, Omeka and the SolrSearch pluginn (use tags for the categories) will get you pretty far down the road.

    If you want to go the Ruby route, use the rsolr gem (http://rubygems.org/gems/rsolr) of the many possibilities. Using dynamic fields, you should be able to arbitrarily add repeatable fields. Add a pointer to image files(s), with indexed text fields, you should be able to quickly prototype a system for this. Solr also has a great facility with the ExtractingRequestHandler (http://wiki.apache.org/solr/ExtractingRequestHandler) to pull full-text out of PDFs, XML, HTML, Word documents, etc.

    With PHP, I prefer the Solr PHP Client (http://code.google.com/p/solr-php-client/) to some of the other approaches.

    With some proper citations, I think you could build this system pretty quickly...

    Hi,

    I think you'd benefit from using Microsoft Expression Media, formerly known as iView MediaPro (before Microsoft bought it). It depends, though if this is for personal reference or if you want to create an online system that provides access to your images and metadata.

    Expression Media allows you to create a catalog of images on a filesystem and enables you to record metadata for each image in your catalog. You can export this catalog into XML, which can then be transformed into a standard of your choosing if you wish to import it into a database or other online system.

    The U.Va. Special Collections library uses this software in their digitization workflow. Metadata recorded about each manuscript in a collection, for example, receives information about its container and its parent as well as a brief description. Metadata records from the Expression Media proprietary XML schema are transformed into MODS records or could be made into Encoded Archival Description finding aids.

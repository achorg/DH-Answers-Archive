    Most of the resources I'm aware of are language-specific; often geared for programmers who want to switch from one technology to another. The Ruby and Ruby on Rails communities have put a lot of effort into eliminating barriers for new learners, however, including building websites and tutorials that allow learners to start experimenting with the technology within their browsers -- without having to install an application framework at all. The two I'm aware of are TryRuby and Rails for Zombies. You can find other Ruby-specific resources at Amanda Wagener's community-contributed directory iwanttolearnruby.com.

    One last suggestion -- it's usually best to have a project in mind when you're working on these things, rather than learning programming in the abstract.

  4. Patrick Murray-John

    I've responded to similar questions in the same way before, so forgive me being a broken record. But at least I'm consistent!

    The best way to sharped coding skills is to code, just as the best way to sharped writing skills is to write.

    So, I'd say look for little etude-like projects that'll push you to learn more to accomplish a task.

    So start with wanting to work with XML and dive into doing some basic manipulation. Especially if your XML experience involves TEI, maybe dive into the New Variorum Shakespeare Digital Challenge data.

    Some possible little things to do might be:

    • Count how many lines each speaker has
    • Calculate the average length of each speakers' lines
    • See which characters use the longest or shortest words

    These likely won't win you the $500 prize, but so what? They, or things like them, might be fun little coding exercises to sharpen your skills. You might even discover that, after you hit on a couple that sharpen some skills, you discover -- and push -- your own coding boundaries.

    The above is just one suggestion for how to get at that principle -- to sharpen your coding skills, put the books and tutorials down and write some code that does small, quick-gratification tasks. Turn back to the tutorials only when you're stuck in how to proceed with writing the code.

    HTH

    This is a bit more meta, but Pragmatic Thinking and Learning: Refactor Your Wetware is a great book on training your brain to think like computers (and write better software).

    I always encourage people to also read as much code as they possibly can. I wouldn't start with framework code, but a nice library for something that you want to do. You mentioned a page turner, so you may want to take a look at the code from this project: http://jpageflipper.codeplex.com/. I would also be sure to get Doug Crockford's JavaScript: The Good Parts and Marjin Haverbeke's Eloquent JavaScript.

    If you're interested in how single-page web applications work, I'd check out Ember.js and Knockout.js.

    If you're looking at the Web Stack, Julie Meloni's books are great. And, if you want to build video games in HTML 5, you can always check out my latest book.

    HTH,
    Wayne

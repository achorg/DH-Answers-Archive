    So I'm tinkering with the TEI in my first batch of "Celestial Railroad" editions (http://celestialrailroad.org/), and I'm looking for the best way to indicate, in the metadata, a few features of each text. I want to be able to indicate the denomination and/or political affiliations of each newspaper/magazine that reprinted the story. I thought I could add some fields in the <sourceDesc>: e.g. <sourceDenomination> or <sourcePolitics>, but that seems to raise red flags when I attempt to post or use the TEI file. Any suggestions for how I can encode this information? The religious and/or political backgrounds of the newspapers are central to the questions I'm asking about these texts.

  3. Patrick Murray-John

    Adding new elements to TEI, at least in a non-uncouth way, generally means extending the TEI itself http://www.tei-c.org/Guidelines/Customization/odds.xml . Of course, it is possible to do what you want and ignore the warnings about invalid markup, but then you still have the problem of teaching whatever tools you are using about the new elements.

    One way at it is to make liberal use of the @type attribute:

          <sourceDesc>
        <listPerson>
          <org>
            <name>Society of Silly Walkers</name>
            <note type="politics">silly</note>
            <note type="denomination">Pythonian</note>
          </org>
        </listPerson>
      </sourceDesc>

    You still have to teach whatever tools you're using for processing what to do with the type attribute, but that keeps things within TEI's framework, and so minimizes that effort.

    (TEI validation can sometimes be odd when you aren't including just a <p>, so it might take some experimentation to get rid of validation flags)

    HTH

    So you're thinking future inter-operability. Then Patrick's advice to suggest it to the TEI folk is the best way to go here. Your work brings up an interesting issue about the organic growth of TEI. If scholars will keep finding categories that will be relevant for large bodies of material, will we ever stop adapting the TEI? It seems on the surface that the amount of categories that are generalizable and applicable to large archives are finite, but there is that nagging question in the back that shifting concerns in the disciplines will generate an unending number of new categories. If that is the case, then the task would be to figure out a way to incorporate those in the thinking of TEI so that those things can be handled at the attribute level.

    I'm sure, though, that these things have been part of the discussions for years now, so I guess all I'm saying is "we're still not there yet."

  8. mudshark

    In the source description mostly you just describe the source/s from which your TEI document is transcribed, using fairly conventional bibliographic fields like author, title, publication date etc. Few bibliographies would characterise whole journals as "left wing" or "rosicrucian" or whatever terms your desired taxonomy would contain. If you're trying to characterise the document in that sort of way , have you considered using one of the children of <textClass> like <catRef> or <keywords>?

    "suggest it to the TEI folk" is definitely a good suggestion (that's what the TEI-L list is for)... as long as you remember that we're all TEI folk! The standard develops and evolves organically responding to the changing requirements of its community. I don't expect those requirements to ever stop changing, any more than I expect humanities scholarship to find the One True Answer and go home.

    I am interested in projects that have used (relatively light) structural TEI to markup issues of a periodical. There isn't an example of such a case in the excellent TEI by Example site nor could I find nice documentation elsewhere.

    Chiefly I'd like to be able to markup individual items within each issue so that a query could return all material by one author, or a list of all items which are "reviews" or "prose" or "poetry" and so on. When I tried to figure out the best way to markup a table of contents for a single issue of a periodical, for example, I wasn't able to find too many resources (largely, I think, b/c my google-fu failed; the query "TEI 'table of contents'" returns, predictably, lots of TEI documentation which itself has a table of contents).

    Any suggestions or guidance for readings would be very welcome.

  2. mdalmau
    Member

    At Indiana University, we encoded following the TEI P4 Guidelines, our first scholarly journal, The Indiana Magazine of History (IMH, http://www.dlib.indiana.edu/collections/imh/) back in 2008. The journal is encoded at the issue-level with article-level bibliographic metadata stored in independent headers, which we use for querying author, title, etc. Since we encode at the issue level, article types are designated in the divs (e.g., reviews) and are used as search filters. The site makes available an "on-the-fly" TEI file at the article level. If you'd like to see what our issue-level encoding looks like, email me (mdalmau@indiana.edu) and I'll send you a file.

    You can find some basic information about our encoding approaches on the web site (http://webapp1.dlib.indiana.edu/imh/projectInfo/encoding.do), but because my colleagues and I, like you, found very little guidance for how to best model the markup for serials for which we wanted both browseable issue-level access and searchable article-level access, two of us published an article that includes greater detail of our encoding practice and discovery mechanisms:

    Dalmau, Michelle and Melanie Schlosser. "Challenges of Serials Text Encoding in the Spirit of Scholarly Communication." Library Hi Tech 28. 3 (2010): 345-359.

    I could also share with you our internal IMH encoding guidelines, but they are not for the faint-hearted.

    Replying to @cforster's post:

    One approach is to create a TEI document for each article. You can then easily attach metadata to each article using the teiHeader. You would need to come up with a way to tie together the articles in an issue. There are various approaches here, some using TEI and some using an outside method like METS.

    As for encoding a table of contents, see http://www.tei-c.org/release/doc/tei-p5-doc/en/html/DS.html#DSFRONT , which discusses use of type="contents" on a div. As explained here, this div would contain a <list> (which would contain an <item> for each entry in the TOC).

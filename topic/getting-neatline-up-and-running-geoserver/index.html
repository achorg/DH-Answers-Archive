  4. Wayne Graham

    Best Answer

    It's possible to do this, though not recommended. Basically you'll need the the IP address of your work computer and pass that to the NeatlineMaps plugins (e.g. 192.168.1.101:8080/geoserver). This can cause some issues because most likely the IP address of your work computer is dynamically allocated and can change, so you'd need to be updating the location of your Geoserver from time-to-time. There are also institutions that block traffic on certain ports (including port 8080, the default Tomcat port). There are ways around this, but you're getting in to some pretty advanced configuration.

    Setting up Tomcat can be quite a pain (I'm not even sure if it's available on Dreamhost), but once you have it running, you can just drop the Geoserver war in your webapps directory. I will note that I'm working with a hosting company (Acugis) right now for a one-click installer for Omeka, Neatline, and Geoserver. I was able to set this up even without it in about 5 minutes (but I have a lot of scripts to automate this since I do it so often) on their system.

    If you do have a spare "server" computer you can use in your office, be sure to check out the OpenGeo suite community edition (http://opengeo.org/products/suite/community/) which makes installing/managing Geoserver really simple. I'm running this on both Ubuntu and CentOS and it works like a champ (only install the opengeo-geoserver package though).

    Wayne

  7. Wayne Graham

    Shawn,

    I've used James' AMI on Amazon, and it works like a champ. There's also the OpenGeo Cloud edition, but it starts out around $95/month. One thing about AWS though, it can be a bit confusing, and gets expensive pretty quick if you're doing anything other than your own personal stuff. We have been helping (mostly graduate students) set up James' AMI on EC2 accounts on the dev tier. The awesome thing is that RDS has a dev tier as well, so you don't have to run your database locally.

    VPS solutions are great, and I've used both linode (http://www.linode.com/) and Rackspace (http://www.rackspace.com/). The nice thing is that you can (generally) very quickly add more memory/cpu power if you need it.

    The tradeoffs here are in the administration of the system. If you're comfortable with configuring and tuning Tomcat, VPS or AWS are a good choice. If you would just as soon not worry about what -XX:+UseParallelGC does and let someone else worry about that, the hosted solution is a great option.

    James, great AMI by the way. Saves a lot of time remembering the various PHP package names that need to be installed...

    To make a quick and partial start to an answer to this great question: I can chime in with positive experiences with MAMP (http://www.mamp.info/en/index.html) and, less extensively, WAMP (http://www.wampserver.com/en/) for Drupal, Wordpress, and Omeka local installations.

    Networks: This past quarter I worked on network visualizations with students on Mac and Windows, and Gephi was frustratingly buggy for students on both (UI problems and numerous instances of crashes and spontaneously lost data). I'm considering teaching students to use D3.js (d3js.org) in the future, which gets students down and dirty in JavaScript with about the same learning curve as Gephi's UI. (D3 doesn't have all the analytics built-in, but there are packages out there for running those analytics in python - see http://www.jgoodwin.net/?p=1223 for one D3 workflow visualizing journal co-citations.)

    Finally, would installing a Linux distribution (say, Debian) through VirtualBox be cheating? That might be useful for students wanting to learn ruby and python in a "sandbox" on a Windows machine. The clone-ability of the virtual machines/states seems like a plus, from an administrative standpoint (for keeping uniform versions of ruby and gems, especially).

    Looking forward to seeing others' answers! Scott

    Software and hardware requirements will change over time, so physical space may be the more important consideration. I recommend a central laptop "hub" on movable tables (don't forget outlet access), and large screen workstations, whiteboards, and projection areas along the walls.

    I have used XAMPP (http://www.apachefriends.org/en/xampp.html) for PHP/MySQL development very successfully. It also runs Python, though I have chosen to go with a separate installation using Anaconda (http://docs.continuum.io/anaconda/index.html). There is a 64-bit version, and it has an impressive array of pre-installed packages useful for DH, some of which can be tricky to install on your own. My only complete is that the wonderful Spyder IDE (https://code.google.com/p/spyderlib/)--worth installing on its own--tends to crash occasionally in the Anaconda environment. But a little Googling revealed that deleting the temporary file folder restores functionality, so I'm not too bothered.

    I would also recommend R Studio (http://www.rstudio.com/), and impressive IDE for R (see also the web-development version Shiny in the nav menu).

    Don't forget Git/GitHub for Windows!

    Scott

    I've listed some popular software pieces in this DH guide, and I would also highly recommend Sublime Text 2 for a text editor. Oxygen is worth it if you're doing any XML encoding work, and I would also suggest ABBYY or another OCR software being available in the lab. And don't forget about Bamboo DiRT for finding, evaluating, and reviewing digital research tools. Good luck!

  3. justinlittman

    Replying to @kdonovan11's post:

    The Twitter API supports retrieving the last 3,200 tweets from a user's timeline. A commandline tool like Twarc (https://github.com/docnow/twarc) can be used to get the user's timeline from the API. If you're looking for web interface, Social Feed Manager (https://gwu-libraries.github.io/sfm-ui/), which I work on, is another option.

    Options for getting historical tweets for a hashtag are much more limited. The Search API only indexes the last 7-9 days. The search on Twitter's website indexes further back in time, but the nature of what is indexed (and not indexed) is unknown.

    Data service providers such as DiscoverText (https://discovertext.com/) allow for greater access to historical data, but obviously at a cost.

    Replying to @justinlittman's post:

    Thanks Justin. DiscoverText (https://discovertext.com) is an option for day-forward Twitter via the Search API or using the Gnip PowerTrack. For historical Twitter, people need to use Sifter (https://sifter.texifter.com) to generate up to three free estimates per day. If there is an estimate you like, there is a price to access the data using DiscoverText.

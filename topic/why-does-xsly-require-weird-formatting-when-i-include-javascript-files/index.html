    This shouldn't really be specific to XSLT, but basically the HTML schema requires a text node (PCDATA) as a child of script (i.e. it can't be empty). See also http://aspnetresources.com/blog/empty_vs_nonempty_html_tags

  3. Joe Gilbert
    Admin

    Just to add to St√©fan's answer, most XSLT processors will convert

    <script src="whatever.js"></script>

    to a self-closing element. Since browsers don't expect the tag to be empty, they fail to render everything after the tag (often the entire visible part of the page).
